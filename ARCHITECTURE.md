## ğŸ¯ Design Patterns Used

### 1. **Abstract Factory Pattern**
- Base classes: `VectorStore`, `LLM`, `Embedding`, `Chunker`
- Concrete implementations can be swapped via configuration

### 2. **Strategy Pattern**
- Different chunking strategies (Fixed, Semantic)
- Different LLM providers (OpenAI, HuggingFace)

### 3. **Builder Pattern**
- `IngestionPipeline` builds the ingestion workflow
- `RAGPipeline` builds the generation workflow

### 4. **Dependency Injection**
- Components receive dependencies via constructor
- Easy testing with mocks

### 5. **Repository Pattern**
- `VectorStore` abstracts data storage
- Unified interface for different backends
